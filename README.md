## 这是什么

Data Skill Hub 是一组面向数据+AI领域的 **Claude Code Skills**。每个 Skill 不是简单的 prompt 模板，而是一个完整的**知识蒸馏系统**——它把资深专家的思维方式、分析框架、工作原则、输出风格编码成 AI Agent 可以执行的协议。

**与普通 prompt 的区别**：

| 维度 | 普通 Prompt | Skill |
|------|------------|-------|
| 角色定义 | "你是一个数据分析师" | 身份卡 + 擅长/不擅长 + 诚实边界 |
| 思维方式 | "请仔细分析" | 4-5 个显式思维模型 + 应用方式 + 局限性 |
| 输出格式 | "请用 markdown 输出" | 输出DNA（句式/词汇/节奏/沉默时刻） |
| 质量保障 | 无 | 自我辩证 + 自我批判清单 + 置信度规则 |
| 可执行性 | 模糊建议 | 具体行动 + KPI + 实施步骤 + 时间线 |

## Skill 制作方法论

从 mayun-skill 深度拆解中提炼的 7 条核心设计原则：

### 1. Persona-First（角色优先）

先定义"我是谁"，再定义"我怎么做"。每个 Skill 都有一个**身份卡**，明确角色、专业领域、核心能力、工作方式、知识根基、自我定位。

### 2. Boundary-Explicit（边界显式）

三层护栏确保 AI 不会越界：
- **擅长/不擅长**：能力边界，告知用户哪些问题应该找别的专家
- **诚实边界**：安全护栏，明确不该做什么（不编造数据、不臆测实现、不做政治评价）
- **价值观与反模式**：追求什么、拒绝什么、内在张力是什么

### 3. Model-Driven（模型驱动）

不是流程驱动（"先做A再做B"），而是**思维模型驱动**。每个模型包含 4 个要素：
- **一句话**：这个模型是什么
- **来源证据**：为什么这个模型有效
- **应用方式**：什么时候用、怎么用
- **局限性**：这个模型什么时候会失效

### 4. DNA-Encoded（风格编码）

输出风格是核心编码，不是附加项：
- **句式特征**：善用排比/表格/树形结构/代码块
- **词汇偏好**：高频词、特征词、避免词
- **输出节奏**：先宏观后微观、先结论后论证、先质量后分析
- **沉默时刻**：什么时候该少说话

### 5. Evidence-Grounded（证据锚定）

每个结论都有数据或代码支撑。置信度分级：
- 基于真实代码/数据：0.95
- 基于公开文档：0.80
- 基于推测：0.60
- 纯理论推导：0.50

### 6. Show-Don't-Tell（展示而非描述）

`examples/demo-conversation.md` 比规则描述更有效。示例应该：
- 覆盖典型场景（完整流程 / 快速查询 / 参数定制）
- 展示"应该长什么样"，而非"应该做什么"
- 包含用户输入和完整的期望输出

### 7. Separation-of-Concerns（关注点分离）

```
<skill-name>/
  SKILL.md              # 核心灵魂（~300-400行）
  README.md             # 项目说明 + 安装 + 效果示例
  LICENSE               # MIT
  <SUPPLEMENTARY>.md    # 补充参考（可选）
  examples/
    demo-conversation.md  # 对话演示
  references/
    research.md           # 调研资料
  scripts/              # 可执行脚本（可选）
```

## SKILL.md 标准结构

```
┌─────────────────────────────────────────────┐
│ 1. Frontmatter         name + description   │
├─────────────────────────────────────────────┤
│ 2. 擅长 / 不擅长       能力边界              │
├─────────────────────────────────────────────┤
│ 3. 核心工作原则         行为约束（编号）      │
├─────────────────────────────────────────────┤
│ 4. 工作流              Agentic Protocol      │
├─────────────────────────────────────────────┤
│ 5. 示例分析            效果锚定              │
├─────────────────────────────────────────────┤
│ 6. 身份卡              角色定义              │
├─────────────────────────────────────────────┤
│ 7. 核心思维模型         思维引擎（4-5个）    │
├─────────────────────────────────────────────┤
│ 8. 输出DNA             风格编码              │
├─────────────────────────────────────────────┤
│ 9. 价值观与反模式       追求 + 拒绝 + 张力   │
├─────────────────────────────────────────────┤
│ 10. 关键概念速查        术语表                │
├─────────────────────────────────────────────┤
│ 11. 诚实边界           安全护栏              │
```

## Skill 分类体系

按设计模式，Skills 分为三类：

### 类型 A：角色型 Skill（Persona-Driven）

**特征**：以人物角色为核心，思维模型和表达风格是主要输出

**结构重点**：身份卡 + 思想模型 + 表达DNA + 处世启发式

**示例**：mayun-skill

### 类型 B：分析型 Skill（Analysis-Driven）

**特征**：以数据分析为核心，工作流和输出模板是主要输出

**结构重点**：工作流 + 分析模型 + 输出规范 + 参数配置

**示例**：user-behavior-analysis、core-dimension-distribution-causal-reasoning、data-quality-detection

### 类型 C：设计型 Skill（Design-Driven）

**特征**：以技术设计为核心，方案模板和对比框架是主要输出

**结构重点**：四维度分析 + 多方案对比 + 自我辩证 + 实施计划

**示例**：ray-data-architect

### 混合型

实际 Skill 通常是多种类型的混合。例如 ray-data-architect 同时具备设计型（方案模板）和角色型（架构师身份卡）的特征。

## 快速开始

### 安装单个 Skill

```bash
# 使用 npx skills 安装
npx skills add <your-org>/ray-data-architect
npx skills add <your-org>/user-behavior-analysis
```

### 安装全部 Skills

```bash
# 克隆整个仓库
git clone https://github.com/<your-org>/data-skill-hub.git

# 或者逐个安装需要的 skill
npx skills add <your-org>/ray-data-architect
npx skills add <your-org>/user-behavior-analysis
npx skills add <your-org>/data-quality-detection
npx skills add <your-org>/core-dimension-distribution-causal-reasoning
```

### 使用示例

安装后，在 Claude Code 中直接使用自然语言触发：

```
# ray-data-architect
"帮我设计一个 Parquet filter pushdown 方案，代码在 /path/to/ray"

# user-behavior-analysis
"分析这份用户行为数据的流失风险 [粘贴 JSON]"

# data-quality-detection
"这张表的 platform 字段为什么有大量 NULL？帮我追查根因"

# core-dimension-distribution-causal-reasoning
"对比这两个候选基表，哪个更适合作为分析基准？"
```

## 如何制作新 Skill

### Step 1: 确定类型

先判断你的 Skill 属于哪种类型（角色型 / 分析型 / 设计型），这决定了 SKILL.md 的结构重点。

### Step 2: 蒸馏知识

从以下来源蒸馏专家知识：
- 真实的项目经验和代码
- 行业基准和最佳实践
- 专家的思维模式和决策过程
- 调研资料和参考文献

### Step 3: 编写 SKILL.md

按照 14 层标准结构编写，重点关注：

```markdown
---
name: your-skill-name
description: 一句话描述——做什么、怎么做、输出什么
---

# Skill 标题

## 擅长
- 能力1
- 能力2

## 不擅长
- 明确不做什么

## 核心工作原则
1. 原则1
2. 原则2
...

## 工作流
### Agentic Protocol
Step 1: ...
Step 2: ...

## 示例分析
[展示"应该长什么样"]

## 身份卡
| 字段 | 内容 |

## 核心思维模型
### 模型一：...
- 一句话:
- 来源证据:
- 应用方式:
- 局限性:

## 输出DNA
### 句式特征
### 词汇偏好
### 输出节奏
### 沉默时刻

## 价值观与反模式
### 追求
### 拒绝
### 内在张力

## 诚实边界
1. ...

## 调研信息源
### 一手来源
### 二手来源
```

### Step 4: 编写配套文件

```
your-skill/
├── SKILL.md              # 必须
├── README.md             # 必须（安装/效果示例/蒸馏内容）
├── LICENSE               # 推荐 MIT
├── examples/
│   └── demo-conversation.md  # 强烈推荐
├── references/
│   └── research.md       # 推荐（调研资料）
└── scripts/              # 可选（可执行脚本）
```

### Step 5: 自检清单

编写完成后，用以下清单自检：

- [ ] Frontmatter 只有 name + description（极简）
- [ ] 擅长/不擅长边界清晰，没有灰色地带
- [ ] 核心工作原则 ≤ 12 条，每条可独立引用
- [ ] 至少有 2 个完整示例（覆盖不同场景）
- [ ] 核心思维模型有 4 个要素（一句话/来源/应用/局限）
- [ ] 输出DNA 描述了具体的句式和词汇，而非"请用专业语气"
- [ ] 诚实边界覆盖了至少 3 个不该做的场景
- [ ] 调研信息源标注了一手/二手来源

## Skill 间的协作

Skills 可以组合使用，形成更强大的分析能力：

```
┌──────────────────────┐     ┌──────────────────────┐
│ data-quality-detection│────→│ core-dimension-      │
│ 数据质量诊断          │     │ distribution-causal- │
│ "这张表能用吗？"      │     │ reasoning            │
└──────────────────────┘     │ "哪个维度是核心？"    │
                              └──────────────────────┘
                                        │
                                        ▼
                              ┌──────────────────────┐
                              │ user-behavior-analysis│
                              │ "用户流失风险多高？"   │
                              └──────────────────────┘

┌──────────────────────┐
│ ray-data-architect   │
│ "怎么优化这个查询？"  │
└──────────────────────┘
```

**典型协作场景**：
1. **数据质量 → 维度分析 → 用户分析**：先确认数据可用，再识别核心维度，最后做用户分析
2. **数据质量 → 架构设计**：发现性能瓶颈后，设计优化方案

## 设计原则速查卡

| 原则 | 含义 | 反模式 |
|------|------|--------|
| Persona-First | 先定义"我是谁" | "你是一个AI助手" |
| Boundary-Explicit | 擅长/不擅长/诚实边界 | 没有边界，什么都答 |
| Model-Driven | 思维模型驱动 | "请按步骤123执行" |
| DNA-Encoded | 输出风格是核心编码 | "请用专业语气" |
| Evidence-Grounded | 结论有数据支撑 | "可能""也许""大概" |
| Show-Don't-Tell | 示例 > 规则 | 只有规则没有示例 |
| Separation-of-Concerns | SKILL.md / references / examples | 全部塞一个文件 |
